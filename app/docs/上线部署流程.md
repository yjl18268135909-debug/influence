# ShopFluence 上线部署流程

目标：把当前本地版上线为公司 5-10 人可同时使用的版本。

## 1. 准备 Supabase 数据库

1. 打开 Supabase 项目。
2. 进入 SQL Editor。
3. 打开本项目文件 `supabase/production_schema.sql`。
4. 全选复制 SQL，粘贴到 Supabase SQL Editor。
5. 点击 Run。

注意：这份 SQL 会重建旧系统兼容表结构，适合正式上线前的初始化数据库。

## 2. 获取 DATABASE_URL

1. Supabase 左侧进入 Project Settings。
2. 进入 Database。
3. 找到 Connection string。
4. 选择 URI 格式。
5. 复制连接字符串。
6. 把其中的 `[YOUR-PASSWORD]` 替换成创建项目时设置的 Database Password。

格式大概类似：

```text
postgresql://postgres.xxx:你的数据库密码@aws-xxx.pooler.supabase.com:6543/postgres
```

## 3. 导入旧数据

在本地终端进入项目：

```bash
cd "/Users/yaojiali/Documents/New project/app"
DATABASE_URL="你的 Supabase 数据库连接字符串" npm run migrate:prod
```

看到每张表输出导入行数，并出现 `Migration completed.` 就代表旧数据已导入云数据库。

## 4. 本地连接云数据库测试

```bash
cd "/Users/yaojiali/Documents/New project/app"
DATABASE_URL="你的 Supabase 数据库连接字符串" npm run server:start
```

再打开：

```text
http://localhost:5001/api/health
```

返回里的 `database` 应该是 `postgres`。

## 5. 部署到 Render

1. 打开 Render。
2. New + 选择 Web Service。
3. 连接这个 GitHub 仓库。
4. Root Directory 填：

```text
app
```

5. Build Command 填：

```bash
npm install && npm run build
```

6. Start Command 填：

```bash
npm start
```

7. Environment Variables 添加：

```text
NODE_ENV=production
DATABASE_URL=你的 Supabase 数据库连接字符串
DATABASE_SSL=true
```

8. 点击 Deploy。
